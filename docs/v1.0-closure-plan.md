# Flame v1.0 Closure Plan

Date: 2026-04-05

## Goal

Close Flame v1.0 with three final steps: stabilize `ssh` in the TUI, modernize `rev`, then finish help and README polish.

## Step 1: SSH

- support `ssh user@ip -p pass`
- support `ssh user@ip -i key`
- run the SSH handoff in the background from inside Flame
- wait for the reverse shell to connect back
- fail clearly if `sshpass` is missing or the shell times out

## Step 2: Payloads

- keep plain `rev` as the payload dump mode
- add `php` generation
- make `rev bash`, `rev ps1`, `rev csharp`, and `rev php` copy to clipboard by default
- keep `rev csharp file.exe` and `rev php shell.php` as file output modes

## Step 3: Help + README

- finalize terminal help
- update README usage and examples
- freeze the v1.0 surface after docs and behavior match

## Boundary

Anything else becomes v1.1+
